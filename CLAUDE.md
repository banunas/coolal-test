# 프로젝트 개발 규칙

우리는 원활한 협업과 안정적인 서비스 운영을 위해 아래의 규칙을 준수합니다.

## 개발 원칙
- 모든 기능은 구현 전 테스트 코드를 먼저 작성합니다.
- 변경 사항은 가능한 작게 나누어 빈번하게 병합합니다.
- 모든 코드는 메인 브랜치에 직접 또는 짧은 수명의 브랜치를 통해 지속적으로 통합합니다.

## 기술 및 환경 규칙
- 코드 들여쓰기는 탭(tabs)을 사용합니다.
- Spring과 React를 사용하여 개발합니다.

## 코드 관리 및 커밋
- 커밋 메시지 작성 시 깃허브 CONTRIBUTING.md의 가이드를 엄격히 준수합니다.
- Always activate the virtual environment using 'source venv/bin/activate' before executing any Django management commands.
