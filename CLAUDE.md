# 프로젝트 개발 규칙

## 코드 작성 원칙
- 우리는 모든 코드를 작성하기 전에 테스트 코드를 먼저 작성하는 TDD 방식을 따른다.
- 모든 코드는 4칸의 스페이스를 사용하여 들여쓰기한다.
- 기술 스택으로 Spring 프레임워크를 기반으로 개발한다.

## 코드 관리 및 협업
- 우리는 모든 변경 사항에 대해 코드 리뷰를 필수로 거친다.
- 모든 커밋 메시지는 Conventional Commits 명세에 맞춰 작성한다.
- 변경 단위는 가능한 작게 유지하여 Small PR을 지향한다.
- 코드 변경 시 관련 기능의 문서화를 필수로 진행한다.
- 프런트엔드 타입 검사 시 항상 'cd ~/Desktop/week2/frontend && npx tsc --noEmit' 명령을 사용하고, 완료 후 'rm -f tsconfig.tsbuildinfo'로 빌드 파일을 정리해 주세요.
