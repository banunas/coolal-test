# 프로젝트 개발 규칙

우리는 다음 규칙을 준수하여 프로젝트를 진행한다.

## 개발 원칙
- 모든 기능 구현 전에 테스트 코드를 먼저 작성한다.
- 변경 사항은 가능한 작게 나누어 커밋하고 풀 리퀘스트를 생성한다.

## 코드 스타일 및 환경
- FastAPI를 사용하여 서비스를 개발한다.
- 들여쓰기는 스페이스 4칸을 사용한다.

## 운영 원칙
- 팀 프로젝트이지만, 기능 구현 및 코드 작성은 개인의 책임하에 독립적으로 진행한다.
- 프론트엔드 타입 검사 후에는 항상 tsconfig.tsbuildinfo 파일을 삭제해 주세요.
