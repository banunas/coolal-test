# 프로젝트 개발 규칙

## 개발 원칙 및 스타일
- 모든 기능은 테스트 코드를 먼저 작성한 후 구현한다.
- 코드는 4개의 스페이스를 사용하여 들여쓰기한다.
- Python의 PEP 8 스타일 가이드를 준수한다.

## 작업 방식
- 프로젝트의 모든 변경 사항은 나 혼자 책임지고 관리한다.
- 외부 의존성 추가 시 반드시 사전에 충분히 검토한다.
- 코드의 가독성과 유지보수성을 최우선으로 고려하여 작성한다.
- 프론트엔드 변경 사항을 저장하거나 수정할 때마다 `cd ~/Desktop/week2/frontend && npx tsc --noEmit`을 실행하여 타입 에러를 검사하고, 이후 `rm -f tsconfig.tsbuildinfo`를 통해 빌드 캐시를 정리해 주세요.
